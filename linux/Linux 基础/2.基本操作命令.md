- **POSIX 标准**:


#### 5.环境变量初始化

`source`

在**当前** shell 环境中读取和执行指定的脚本文件，该脚本中定义的变量、执行的命令、设置的环境变量等都会直接影响当前的 shell 环境。 经常用于改变当前命令执行的上下文（环境变量，函数，变量）



对于 `Bash` ，加载文件顺序是：

Linux 的环境变量初始化是在用户登录时由 shell 程序处理的一个过程。这个过程涉及到多个配置文件的读取和执行。具体涉及到的文件以及他们的读取顺序取决于你所使用的 shell 类型，最常见的是 Bash 和 Zsh。

### 对于 Bash Shell
当一个用户登录到系统中时（例如通过控制台登录或通过 SSH），Bash shell 会按照下列顺序读取和执行环境设置文件：

```

/etc/profile    --->  ~/.bash_profile  ---> ~/.bashrc  --> /etc/bash.bashrc(/etc/bashrc)  -> ~/.bash_logout
    |                 ~/.bash_login, 
    |                 ~/.profile
    |    
    |
/etc/profile.d/*.sh
```

- 调用顺序是通过在用户的个人配置文件中显式编写代码来完成。不同的登录方式只是入口不同。
- 交互式登录Shell，入口时 `/etc/profile`。交互式非登录 Shell，入口是 /.bashrc。非交互式登录 Shell，Bash 不主动加载上述配置文件。
