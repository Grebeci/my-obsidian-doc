## 定义

对于查询中的每一行，使用与该行相关的行执行计算。

他是为了解决下列问题产生的：

- group by 之后，select 只能引用 key，或者使用聚合函数引用其他的列。这是由于聚合后粒度变粗导致的。窗口函数基于每行进行分析，粒度最小。

	例如 求 每个部门薪资最高的员工（`（key,agg_func(value) -> value` ）的问题需要一次回表操作，原因是group by 后粒度变大。

- 无法引用到前后行的数据，导致无法解决连续问题。



## 概念



标准SQL语法

```
window_function_name() over_clause 
```



大多数的窗口函数是：

1. 聚合函数都能当成窗口函数使用
2. 前后行（ lead，lag，nth_value,first_value, last_value）, 行引用( row_number, rank ,dense_rank)



指定窗口大小：

- 物理窗口 rows，按照当前行位置来算。
- 逻辑窗口 ranges，根据某列的值来算（日期时间）。

注： 缺省指定窗口大小的，取决于 `order by` 子句和具体的窗口函数的行为。

#### 3. 指定窗口大小

```mysql
frame_extent:
    {frame_start | frame_between}

frame_between:
    BETWEEN frame_start AND frame_end

frame_start, frame_end: {
    CURRENT ROW
  | UNBOUNDED PRECEDING
  | UNBOUNDED FOLLOWING
  | expr PRECEDING
  | expr FOLLOWING
}
```

只指定 frame_start 的，隐含了 窗口的结束是 current row



特别的：一些窗口函数不显式指定窗口大小时候。

- 聚合函数（ count, sum, max, min, avg)

	不加 order by 子句   [ UNBOUNDED PRECEDING , UNBOUNDED PRECEDING ]

	有 order by 子句   `[ UNBOUNDED PRECEDING , CURRENT ROW ]`

- rank, row_number, dense_rank

	不加order by ，行编号不确定（语义错误）

	有 order by , `[ UNBOUNDED PRECEDING , CURRENT ROW ]`

#### 窗口大小

When ORDER BY is specified with missing WINDOW clause, the WINDOW specification defaults to `RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW.`

When both ORDER BY and WINDOW clauses are missing, the WINDOW specification defaults to `ROW BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING.`
